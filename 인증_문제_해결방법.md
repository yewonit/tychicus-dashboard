# GitHub 인증 문제 해결 방법

## 문제 상황
```
remote: Permission to yewonit/tychicus-dashboard.git denied to kimwoonbi.
```

현재 로그인된 계정("kimwoonbi")과 저장소 소유자("yewonit")가 달라서 발생한 문제입니다.

## 해결 방법

### 방법 1: Personal Access Token 사용 (권장)

1. **GitHub에서 Personal Access Token 생성**
   - GitHub 로그인 후: https://github.com/settings/tokens
   - "Generate new token (classic)" 클릭
   - Note: "DUGIGO v0.6 push" (설명)
   - Expiration: 원하는 기간 선택
   - Scopes: **`repo`** 체크 (전체 저장소 권한)
   - "Generate token" 클릭
   - **토큰을 복사해두세요!** (한 번만 표시됩니다)

2. **자격 증명 업데이트**
   
   **Windows 자격 증명 관리자 사용:**
   - Windows 검색에서 "자격 증명 관리자" 실행
   - "Windows 자격 증명" 탭 선택
   - `git:https://github.com` 검색 후 삭제
   - 또는 `github.com` 검색 후 관련 항목 삭제
   
   **다시 푸시 시도:**
   ```bash
   git push -u origin v0.6-admin-mockup
   ```
   - Username: `yewonit`
   - Password: **생성한 Personal Access Token 입력**

### 방법 2: Git Credential Manager 사용

```bash
# 자격 증명 저장 설정
git config --global credential.helper manager-core

# 푸시 다시 시도
git push -u origin v0.6-admin-mockup
```

### 방법 3: SSH 키 사용 (장기적으로 권장)

1. **SSH 키 생성** (아직 없다면)
   ```bash
   ssh-keygen -t ed25519 -C "your.email@example.com"
   ```

2. **SSH 키를 GitHub에 추가**
   - 공개 키 복사: `cat ~/.ssh/id_ed25519.pub`
   - GitHub → Settings → SSH and GPG keys → New SSH key

3. **원격 저장소 URL을 SSH로 변경**
   ```bash
   git remote set-url origin git@github.com:yewonit/tychicus-dashboard.git
   ```

4. **다시 푸시**
   ```bash
   git push -u origin v0.6-admin-mockup
   ```

### 방법 4: GitHub Desktop 사용

1. GitHub Desktop 설치: https://desktop.github.com/
2. 저장소 열기: File → Add Local Repository
3. 프로젝트 폴더 선택
4. "Publish branch" 클릭

✅ 새 브랜치 생성 완료 (v0.6-admin-mockup)
✅ 파일 추가 및 커밋 완료 (53개 파일, 42,967줄 추가)
