## 해결 방법 (우선순위순)

### 방법 1: GitHub Desktop 사용 (가장 쉬움) ⭐ 권장

1. **GitHub Desktop 설치**
   - https://desktop.github.com/ 다운로드 및 설치
   
2. **yewonit 계정으로 로그인**
   - GitHub Desktop 실행
   - File → Options → Accounts
   - yewonit 계정으로 로그인

3. **저장소 열기**
   - File → Add Local Repository
   - 프로젝트 폴더 선택: `C:\Users\Kim Woonbi\OneDrive\바탕 화면\DUGIGO_v0.6`

4. **브랜치 푸시**
   - 상단에 `v0.6-admin-mockup` 브랜치가 선택되어 있는지 확인
   - "Publish branch" 버튼 클릭
   - 또는 "Push origin" 버튼 클릭

### 방법 2: 브라우저에서 yewonit 계정으로 로그인

1. **브라우저에서 GitHub 로그아웃**
   - https://github.com 로그아웃

2. **yewonit 계정으로 로그인**
   - https://github.com 에서 yewonit 계정으로 로그인

3. **Git Bash에서 다시 시도**
   ```bash
   cd "/c/Users/Kim Woonbi/OneDrive/바탕 화면/DUGIGO_v0.6"
   git push -u origin v0.6-admin-mockup
   ```

### 방법 3: Personal Access Token 확인

현재 사용 중인 Personal Access Token이:
- **yewonit 계정**으로 생성되었는지 확인
- `repo` 권한이 있는지 확인

토큰이 **kimwoonbi 계정**으로 생성되었다면:
1. https://github.com/settings/tokens 에서 삭제
2. **yewonit 계정**으로 로그인
3. 새 토큰 생성 (`repo` 권한)
4. 새 토큰 사용

### 방법 4: Git Bash에서 직접 입력

```bash
cd "/c/Users/Kim Woonbi/OneDrive/바탕 화면/DUGIGO_v0.6"

# 원격 저장소 URL 확인 및 변경
git remote set-url origin https://yewonit@github.com/yewonit/tychicus-dashboard.git

# 푸시 (인증 프롬프트가 나타나면)

# Username: yewonit
# Password: [Personal Access Token 입력]
```
